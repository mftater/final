Collaborative Calendar Project Proposal

Developer

Mason Tate (Sole Developer)

Project Overview

An interactive JavaScript calendar app that allows users to select custom months and years. The next development phase will introduce shared functionality—allowing family members or spouses to contribute to each other’s calendars, without deleting or modifying existing entries.

Features

Completed / Current

  • Dynamically renders monthly calendars with day-of-week alignment; meaning that the calendar will not have empty cells, this aligns with traditional calendars.
  • User selects year and month from dropdown menus
  • Manual date logic using vanilla JavaScript
  • Basic responsive structure (in progress)

In Progress / Planned

  • Family/Spouse Sharing: Users can view each other's calendars and add new events, but cannot delete others’ events
  • Addition of user profile logic, users can create their own profile to manage calendars, this profile can then approve family member requests (complete, need family integration)
  • Users recieve alerts for additions to calendars or new family member requests (50%, need family connections)
  • Event attribution (e.g., "Added by Mason") (complete)
  • Clickable dates to add events with title, time, and description (complete)
  • Dark Mode toggle for accessibility (complete)
  • Support for years beyond 2020 (complete)
  • Local event saving via localStorage (kinda complete, dark mode toggle is saved, need drafts to be saved)
  • Integration of users, events and an access table to manage all calendar data on the backend using MySQL (complete)
  • Admin dashboard to view stats
  • Some sort of security feature in place to prevent users from overloading anothers calendar, maybe limiting additions or making every addition ask for user approval

Target Users

  • Families, spouses, or roommates needing collaborative scheduling
  • Students and professionals who want simple, clean planners
  • Users who prefer minimal UI over cluttered commercial calendar apps

Tech Stack

  • HTML, CSS, JavaScript, PHP
  • Planned: localStorage, backend (MySQL or JSON files)

Unique Aspects

  • Custom-built date logic (including leap years, weekday matching)
  • No libraries or plugins—fully hand-coded
  • User-respectful design: Add-only permissions for shared calendars; users cannot remove any data from another users calendar
