Calendar Proposal Features

Features Document - Event Management System

This document outlines the features of the Calendar App I created

Overview

The system allows users to add events, view their calendar, and share events with family members. All features are working except sharing events and the admin page.

Front-End Features

1. Event Creation

Description: Users can create new events by filling in a form with event details (title, description, date, time).

Status: Working. The event is added to the calendar successfully, with validation ensuring that required fields (title, date, and time) are filled.

2. Event Display

Description: Events are displayed in the user's calendar.

Status: Working. Events are shown correctly on the calendar after being added with accurate date, time, and title.

3. Share Event Checkbox

Description: Users can select a checkbox to share an event with their family members.

Status: Working, but shared events functionality is currently not working. Although the checkbox allows event sharing, shared events are not being saved properly.

4. Family Member Dropdown

Description: Users can select family members from a dropdown list when creating or sharing an event.

Status: Working. The dropdown populates with family members from the back-end, and users can select multiple family members to share the event with via Id.

Back-End Features

1. User Authentication

Description: Users must be authenticated to create events or view their calendar.

Status: Working. Users can log in and access their account. The system ensures that only logged-in users can access protected routes (event creation, calendar view).

2. Event Insertion

Description: Events are stored in the database when a user creates them.

Status: Working. Events are successfully inserted into the `events` table.

3. Shared Events Insertion

Description: If the user selects the option to share an event, it should be saved in the `shared_events` table.

Status: Not Working. Shared events are not being saved to the `shared_events` table despite the form submitting data for family members. Debugging required.

4. Family Connections

Description: Family members who are connected and have an `approved` status are retrieved to show in the dropdown list.

Status: Working. The `family_connections` table is queried to fetch approved family members and display them in the front-end dropdown.

Issues and Next Steps

1. Shared Events Not Saving

Description: Shared events are not being saved into the `shared_events` table, even though the checkbox is checked and family members are selected.

Possible Causes: Seems to be an issue with data getting to the query, the query is correct when checked with phpmyadmin, however it is nonfunctional

Next Steps:

  • Verify query execution for inserting into the `shared_events` table.
  • More styling, dark mode is functional but not the prettiest
  • Currently trying to decide how to allow a parent to have control over a childs calendar

2. Implementation of methods to prevent users from overloading another with requests

3. Implementation of a stats page for admin users
